4.7、项目的分支管理-创建分支

查看分支   git branch(查看本地分支)  git branch -a (查看所有远程端的分支)
创建一个本地分支   git checkout -b 分支名
推送分支文件：git push --set-upstream origin branch1


4.8、项目的分支管理-删除分支
先切换分支到master: git checkout master
仅删除本地分支：  git branch -d branch1
意义上删除远程分支;  git branch -r -d  origin/branch1(虽然在本地查看时已经删除了远程的分支，但实际没有删除)
实际上删除远程分支：git push origin :branch1


4.9、项目的分支管理-合并分支
进入要合并的分支下：git merge 分支名


4.10、项目的分支管理-合并分支出现冲突

文件冲突展示：
dsd合并分支
<<<<<<< HEAD
这是master上的文件
=======
这是merge分支上添加内容
>>>>>>> mergedemo

解决方式：删除包含<>的内容，重新编辑一个新的，保存推上去即可



4.11、git版本管理


回退上一个版本 :  git reset --hard  HEAD^   回退两个版本 ^^ 回退几个版本就几个^            结果：HEAD is now at e9651b0 version1（e9651b0 是一个唯一的版本号）
回退到指定的版本： git reset --hard  HEAD~100      100是上几个版本     或者  git reset --hard  e9651b0(版本号)
